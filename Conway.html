John Conway’s Game of Life

Conway’s game of life is one of the simplest forms of Cellular Automata, meaning a mathematical model where each of the objects in that model is considered a living object and their lives are governed by simple rules. The rules can cover their behaviours of replication, destruction, and other complex behaviours. There are many other forms of Cellular Automata. Like Rule 30 and Biods.

Conway’s game of life is theoretically played on an infinite grid. It’s often addressed as a no-player game because the initial seed is only the thing that’s responsible for the behaviour of the whole system.

When simulating the game of life, implementing an infinite grid is quite tricky because we need to use an advanced data structure called Hash table and there is an algorithm called Hashlife. In our simulations, we use a wrap-around mechanism meaning that the end of one edge is connected to its pair edge.

The Rules of Life

The system is governed by three types of rules.

  • Birth Rule:
    • Any empty cell with precisely three neighbors will become alive in the next generation.
  • Death Rule:
    • Isolation: Any alive cell with less than 2 neighbors will die in the next generation.
    • Overcrowding: Any alive cell with more than 3 neighbors will die in the next generation.
  • Survival Rule:
    • Any alive cell with 2 or 3 neighbors will survive to the next generation.

Life Objects

Life Objects are characterized by their behaviors on this system. We associate them with names that resemble certain things that we see in real life.

This life structure on the left is called a Pulsar. And the on the right, is a depiction of what is assumed to be a real celestial Pulsar.

Names don’t matter because we are the ones giving the names of the structures based on our life experiences. Mathematicians around the world were so interested in studying this system and the behaviors of the objects. And for half a decade since 1970, when the game was invented, people have been finding new behaviors and objects that exhibit different types of interactions.

Here are some examples of life objects.

  • Still lives:
    • Stable structures don’t get changed over the generations.
  • Oscillators:
    • A pattern that flips back and forth between a set number of periods but will never grow or die.
  • Spaceships:
    • A finite pattern that returns to its initial state after several generations (known as its period) but in a different location.
  • Breeders:
    • A pattern that exhibits quadratic growth, by generating multiple copies of a secondary pattern, each of which then generates multiple copies of a tertiary pattern.
  • Explosives:
    • A pattern that can cause chaotic reactions and completely dies out.
  • Guns:
    • A pattern that can emit spaceships in various directions.

And the list goes on.

Our Simulation

Our simulation is written in JavaScript and with a library called p5.js.

p5.js Web Editor

The simulation is fully written in Object-oriented paradigm.

  • Grid object creates an instance of the grid with wrapping mechanism.
  • An object named GameObject stores information about various structures present in the simulations and it can update the grid.
  • According to the standard p5.js setting, the function named draw() continuously loops during the execution and the function named setup() only gets executed during the start of the program.

The algorithm

  • The grid information (whether the cells are alive or dead) is stored in a 2D array.
  • We loop through all the cells and count the alive neighbors.
  • We create a new 2D array whose values are determined by the previously counted number of alive neighbors.
  • Finally, we draw the new cells based on the newly created(updated) 2D array.
